# requirement.md

## 1. 專案概述
本系統旨在為三間「環保回收站」固定店鋪以及動態開設的「外派街站」提供一個中心化的智慧排班與人力成本管理平台。系統由單一管理員（排班者）統籌，支援員工跨店支援、街站時段動態備註、員工出勤意願管理、人力不足預警，以及**當月人事成本即時試算**。後端採用 Supabase 作為資料庫，前端部署為支援友善列印的動態網頁。

## 2. 核心業務邏輯與規則

### 2.1 地點與街站時段備註
* **固定店鋪：** 太和、大元、廣福（皆為同類型之環保回收站）。
* **外派街站：** 員工在當天班型下，若需前往街站支援，需備註街站時段。時段代碼為：`AM`（上午）、`PM`（下午）、`N`（夜間/晚上）。
* **街站呈現格式：** 在店鋪月曆中，若員工當天有去街站，以 `員工姓名 | 班型 (街站時段組合)` 呈現。
  * *範例：* `李四 | B (AM)`（上 B 班，且上午去街站）
  * *範例：* `王五 | Shift M (APM)`（上 M 班，且上午與下午都去街站）
  * *範例：* `陳六 | C (PMN)`（上 C 班，且下午與晚上都去街站）

### 2.2 班型與工時設定 (Shift Types)
系統包含常規班型與 R 系列輪班，部分班型之工時會根據「星期幾」進行動態調整：

| 班型代號 | 適用星期 / 條件 | 時間範圍 | 工時 (小時) | 備註 |
| :--- | :--- | :--- | :--- | :--- |
| **A 班** | 天天適用 | 07:45 - 16:45 | 9.0 | 常規早班 |
| **B 班** | 天天適用 | 10:00 - 19:00 | 9.0 | 常規中班 |
| **C 班** | 天天適用 | 12:30 - 21:30 | 9.0 | 常規晚班 |
| **D 班** | 天天適用 | 09:00 - 18:00 | 9.0 | 常規辦公班 |
| **Shift M** | 天天適用 | 08:00 - 21:30 | 13.5 | 全日班 |
| **R1** | 天天適用 | 09:30 - 13:30 | 4.0 | 短半班 |
| **R2** | 天天適用 | 14:00 - 18:00 | 4.0 | 短半班 |
| **R3 (型態一)** | 星期一、五 | 17:00 - 21:30 | 4.5 | 夜班 |
| **R3 (型態二)** | 星期慢 | 17:30 - 21:30 | 4.0 | 夜班 |
| **R4** | 天天適用 | 14:00 - 21:30 | 7.5 | 中晚班 |
| **R5 (型態一)** | 星期一、五 | 18:00 - 21:30 | 3.5 | 彈性夜班 |
| **R5 (型態二)** | 星期二至四、六、日 | 18:15 - 21:15 | 3.0 | 彈性夜班 |
| **R6** | 天天適用 | 14:30 - 21:30 | 7.0 | 中晚班 |

> ⚠️ **衝突規則：** 同一人在同一天內，只能被指派至一個地點與一種班型，不可重疊排班。

## 3. 功能需求 (Functional Requirements)

### 3.1 員工資料與人事成本管理 (Employee & Cost Management)
* **員工欄位：** 姓名、職級（Supervisor / 正職 / 兼職）、薪資類型（月薪 / 時薪）、薪資數額（例如：月薪 $25,000 或 時薪 $70）。
* **當月人事成本即時看板（Dashboard）：**
  * 在排班畫面上方或側邊，即時顯示**「當月預估總人事成本」**。
  * **計算邏輯：**
    * **月薪員工：** 不論當月排幾班，皆固定加總其月薪數額。
    * **時薪員工：** 根據當月已排班表之「實際工時（`actual_hours`）」進行累加，再乘以其時薪。
  * **動態更新：** 管理員每新增、刪除或修改一個班表格子，人事成本看板會即時重新計算並更新數字。

### 3.2 人員可用性管理 (Availability Interface)
* **意願輸入工作台：** 提供一個類似 Excel 的網頁矩陣介面（縱軸為員工，橫軸為日期）。
* **快速標記：** 管理員可根據員工提供的信息，快速點擊勾選或切換每位員工在當月的「可上班 (Available)」或「不可上班 (Unavailable)」狀態。

### 3.3 智慧輔助排班畫板 (以店舖為核心的月曆視圖)
* **店舖切換標籤 (Tabs)：** 畫面上方提供「太和」、「大元」、「廣福」、「外派街站匯總」切換按鈕。
* **點擊動態編輯：** 點擊特定日期的班型格子，彈出視窗：
  1. **選擇員工：** 系統自動過濾並推薦當天回報「可上班」且「尚未被指派」的員工。
  2. **街站註記（選填）：** 可多選勾選「上午(AM)」、「下午(PM) Foro」、「晚上(N)」。若勾選，系統會自動組合字串（如 APM）並標註在姓名旁。
* **工時自動判定：** 前端需根據「班別」與「排班當天星期幾」，自動判斷並帶入對應工時（特別是 R3 與 R5 班），無需人工計算。
* **人力缺口提示：** 未達各店設定之最低配置時，格子顯示紅色警告，由排班者手動彈性調度。

### 3.4 友善列印與簡易密碼查看
* **一鍵列印：** 優化 CSS (`@media print`)，隱藏控制按鈕與導覽列，自動將當月店鋪月曆格式化為 **A4 橫向 (Landscape)** 滿版，方便張貼於回收站。
* **簡易密碼查看：** 員工使用全店共通的簡易密碼即可登入「唯讀網頁」，切換觀看與列印各店班表，無法修改任何後端資料。

## 4. Supabase 資料庫實體設計 (Schema)

```sql
-- 1. 員工與薪資表
CREATE TABLE employees (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(50) NOT NULL,
    role VARCHAR(20) CHECK (role IN ('supervisor', 'full_time', 'part_time')),
    salary_type VARCHAR(10) CHECK (salary_type IN ('monthly', 'hourly')), -- 月薪(monthly) 或 時薪(hourly)
    salary_rate NUMERIC(10, 2) NOT NULL -- 薪資數額
);

-- 2. 地點表 (店鋪A, B, C, 外派街站)
CREATE TABLE locations (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) NOT NULL
);

-- 3. 員工可用性表 (意願登記)
CREATE TABLE availability (
    id BIGSERIAL PRIMARY KEY,
    employee_id UUID REFERENCES employees(id) ON DELETE CASCADE,
    date DATE NOT NULL,
    is_available BOOLEAN NOT NULL,
    UNIQUE(employee_id, date)
);

-- 4. 班型主檔表
CREATE TABLE shift_types (
    code VARCHAR(10) PRIMARY KEY -- A, B, C, D, M, R1, R2, R3, R4, R5, R6
);

-- 5. 最終班表 (核心：直接記錄當班實際工時，確保變動工時之成本計算百分之百準確)
CREATE TABLE rosters (
    id BIGSERIAL PRIMARY KEY,
    date DATE NOT NULL,
    employee_id UUID REFERENCES employees(id) ON DELETE CASCADE,
    location_id INT REFERENCES locations(id),
    shift_code VARCHAR(10) REFERENCES shift_types(code),
    actual_hours NUMERIC(4, 2) NOT NULL, -- 由前端根據「班別+星期幾」自動計算並寫入 (例如 R3 在週一帶入 4.5, 週四帶入 4.0)
    street_tags VARCHAR(10) DEFAULT NULL, -- 紀錄 'AM', 'PM', 'APM', 'PMN' 等，無則為 NULL
    UNIQUE(date, employee_id) -- 確保同一人一天只會出現在一個地方上一個班
);
